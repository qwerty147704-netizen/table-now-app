## 프로젝트 개요

### 프로젝트 정보

- **프로젝트명**: Table Now
- **앱 유형**: 선결제 방식의 식당 예약 앱
- **예시 도메인**: 카레 프렌차이즈 (코코이찌방야, 아비꼬, 토모토 카레 등)
- **주요 특징**: 예약 시 메뉴를 미리 주문하고 결제하는 선결제 시스템

### 기술 스택

- **프론트엔드**: Flutter (Dart)
- **백엔드**: FastAPI (Python)
- **데이터베이스**: MySQL 8.0
- **인증**: Firebase Auth, Google Sign-In
- **결제**: Toss Payment

## 백엔드 (FastAPI) 작업

### 인증 관련 API ✅

- **담당자**: 김택권 (프로젝트 관리자)
- **엔드포인트**:
  - [O] `POST /api/customer/register` - 회원가입
  - [O] `POST /api/customer/login` - 일반 로그인
  - [O] `POST /api/customer/social-login` - 소셜 로그인
  - [O] `POST /api/customer/link-social` - 계정 통합
  - [O] `GET /api/customer` - 전체 고객 조회
  - [O] `GET /api/customer/{customer_seq}` - 사용자 정보 조회
  - [O] `PUT /api/customer/{customer_seq}` - 사용자 정보 수정
  - [O] `DELETE /api/customer/{customer_seq}` - 사용자 정보 삭제
  - [O] `POST /api/customer/request-password-change` - 비밀번호 변경 요청
  - [O] `POST /api/customer/verify-password-change-code` - 인증 코드 검증
  - [O] `PUT /api/customer/change-password` - 비밀번호 변경
- **유틸리티**:
  - [O] 이메일 서비스 구현 (`EmailService`)

## 중요 사항

### 선결제 시스템

- 예약 시 메뉴를 미리 주문하고 결제를 완료해야 예약이 확정됨
- 결제 완료 후 예약이 확정되는 플로우 구현 필요
- `reserve` 테이블의 `payment_status` 필드로 결제 상태 관리

### 날씨 정보 연동

- 예약 화면에서 날씨 정보를 실시간으로 조회하여 표시
- OpenWeatherMap API를 직접 호출하여 날씨 정보 조회 (DB 저장 없음)
- `GET /api/weather/direct` 또는 `GET /api/weather/direct/single` 엔드포인트 사용
- 예약 시 날씨 정보는 API에서 직접 가져와서 표시 (DB에 저장하지 않음)
- **변경사항 (2026-01-21)**: weather 테이블 제거, 실시간 API 조회 방식으로 변경

### 소셜 로그인

- Google Sign-In 연동
- Firebase Auth를 통한 소셜 로그인 처리
- 소셜 로그인 시 사용자 정보를 `customer` 테이블에 저장

### 데이터베이스 스키마

- 현재 스키마는 `fastapi/mysql/table_now_db_init_v2.sql`에 정의됨 (v2)
- 시드 데이터는 `fastapi/mysql/table_now_db_current_data_v2.sql`에 포함됨
- 카레 프렌차이즈 예시 데이터 포함 (코코이찌방야, 아비꼬, 토모토 카레)
- **변경사항 (2026-01-21)**: weather 테이블 제거, reserve 테이블에서 weather_datetime 컬럼 제거
